# 3Dダンジョン探索型RPG (仮) ゲームデザイン及び実装計画

## 1. ゲームデザイン概要
初期の「Wizardry」にインスパイアされた、1人称視点の3Dダンジョン探索型2DファンタジーRPGです。レトロなUIとシンプルなハック＆スラッシュの要素を持ちます。

### 1-1. 全体仕様
* **プラットフォーム**: Webブラウザ (HTML / CSS / JavaScript を想定)
* **テーマ・グラフィック**: 黒背景にモノクロ（または数色）のワイヤーフレームやドット絵を基調としたレトロスタイル。
* **基本システム**: グリッドベースのダンジョン移動、ランダムエンカウント、ターン制コマンドバトル。
* **対象スコープ (初期版)**: 
  * 1フロアのみ（約20x20マスの薄い壁方式）
  * プレイヤーキャラクターは1人のみ
  * 職業は「戦士」のみ（魔法なし）

### 1-2. 画面レイアウト
画面は大きく4つのペイン（領域）に分割します。

1. **メインビューエリア (左上)**
   * 現在の視点から見える1人称視点のダンジョン（壁、通路、扉など）を描画。
   * モンスター遭遇時には、このエリアにモンスターの画像（またはシンボル）を表示。
   * 各種メッセージ（「○○があらわれた！」「○○のこうげき！」等）を表示するメッセージボックスを最下部にオーバーレイまたは隣接して配置。
2. **モンスターステータスウィンドウ (メインビュー上部)**
   * 「モンスターエンカウント時」にのみ現れるウィンドウ。
   * 出現したモンスターの `NAME`, `HP`, `STATUS` (状態) を表示。
3. **キャラクターステータスウィンドウ (左下)**
   * 常に表示。
   * プレイヤー自身の情報を表示します。
   * `NAME` (名前), `HP` (ヒットポイント), `MP` (マジックポイント。今回は0固定), `AC` (アーマークラス), `STATUS` (SLEEP, DEADなど)。
4. **オートマッピングウィンドウ (右側)**
   * プレイヤーが踏破した範囲のダンジョンマップ（2Dトップダウン視点）を表示。
   * 「薄い壁」のマス目構造を描画し、現在位置と向いている方角をアイコンで表示。

### 1-3. データ設計
#### プレイヤー (戦士)
* **NAME**: [プレイヤーが入力、またはデフォルト名]
* **HP / MAX_HP**: 体力。0になるとゲームオーバー (`DEAD` ステータス)。
* **MP / MAX_MP**: 魔力。今回は魔法を使えないため `0` とするが、将来の拡張のために表示。
* **AC**: アーマークラス。数値が低い（またはマイナス）ほど攻撃を回避しやすい初期Wizardryの仕様を採用、あるいは直感的に数値が高いほど固い仕様にするかを選択。
* **STATUS**: `NORMAL` (正常), `SLEEP` (睡眠・行動不能), `DEAD` (死亡) など。
* **その他内部データ**: 攻撃力、素早さ、経験値、レベル など。

#### モンスター
* スライム、ゴブリンなどの下級モンスターを設定。
* **NAME**, **HP**, **AC**, **STATUS** 等を保持。
* **行動パターン**: 基本的にランダムにまたは一番HPの低い相手へ「こうげき」のみを行う（今回はプレイヤー1人なので単純化）。

## 2. 実装計画 (フェーズ分け)

プロジェクトを以下のステップに分割して、確実に動くものを段階的に組み上げていきます。

### 【フェーズ 1】 プロジェクトセットアップとUIモックアップ作成
1. `C:\Win_tools\Antigravity\Fantasy-RPG` フォルダにWebアプリケーションのベースを作成。
   * 技術スタックの決定（React + Vite なのか、Vanilla HTML/JS なのか）。※今回はコンポーネント管理が容易な **React (Vite)** などを推奨。
2. CSSを用いて、画面を4つの領域（メインビュー、キャラステータス、敵ステータス、マップ）に分割するレイアウトを構築し、ダミーデータで表示を確認する。

### 【フェーズ 2】 ダンジョンマップと移動・オートマッピング機能
1. **マップデータ構造の定義**: 20x20程度のマス目データ（各マスの北・東・南・西に壁があるかないか等のビットフラグデータ）を作成。
2. **オートマッピングの実装**: 右側のウィンドウに、現在位置と踏破済みマスを描画する処理を作成（HTML5 Canvasまたはdiv要素のグリッド）。
3. **移動操作**: キーボード（W/A/S/D または矢印キー）で前進、後退、右ターン、左ターンができるようにし、マップ上の現在位置・向きが更新されるようにする。

### 【フェーズ 3】 3Dメインビュー描画ロジック (疑似3Dワイヤーフレーム)
1. プレイヤーの現在位置と向きから前方のマス（距離1〜3程度まで）を走査。
2. 壁の有無を判定し、HTML/CSS（またはCanvas）を用いてパースのついた壁の線画を描画するロジックを実装。
3. 歩くごとにメインビューの景色が切り替わることを確認。

### 【フェーズ 4】 キャラクター管理とエンカウント処理
1. プレイヤーデータのState（状態）管理を実装し、左下のステータスウィンドウに反映させる。
2. 一歩移動するごとに乱数判定を行い、一定確率でモンスターが出現する「ランダムエンカウント」処理を実装。
3. エンカウント時、画面がバトルモードに切り替わり、メインビュー上部に敵のステータスウィンドウが表示されるようにする。

### 【フェーズ 5】 バトルシステムの構築
1. コマンド選択UI (`たたかう`, `にげる`, `ぼうぎょ`) の実装。
2. 命中判定計算とダメージ計算ロジック（ダイスロールによる乱数加味など）の実装。
3. プレイヤーターンとエネミーターンの交互処理（素早さに応じた順序）。
4. 勝利時の経験値獲得・リザルト処理、敗北時(DEAD)のゲームオーバー処理。

### 【フェーズ 6】 ブラッシュアップと動作確認
1. メッセージログによる戦闘の経過表示の調整。
2. UIの細かいデザイン調整（レトロフォントの導入、枠線のデザインなど）。
3. バレーボールバグ（壁抜けなど）のテストと修正。
