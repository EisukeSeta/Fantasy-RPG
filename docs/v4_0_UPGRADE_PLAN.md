# 平安魔道伝 羅生門編：v4.0 拡張計画書「都の新生と武勲の理」

本文書は、第一章の完結（v3.5）を受け、都をより深く、美しく、そして堅牢なものにするための第四章開発計画を定義する。

## 1. 開発の三連環（フェーズ）

### Phase 1: 【智の防壁】自動検番と理の証明（QA & Testing）
- **都の自動検番 (Auto Audit)**: `scripts/balance_tester.mjs` に「不均衡検知」を実装。期待値が閾値を下回った際に CI/CD（またはローカル検分）で警告を出す仕組み。
- **理の証明 (Vitest 強化)**: 複数ターンにわたる戦闘推移、および状態異常（将来用）の基礎ロジックをテスト。

### Phase 2: 【演武の理】術名の顕現と視覚の響き（VFX & Immersive）
- **言霊の咆哮 (Spell Action Name)**: 戦闘時、術を発動した瞬間にその御名（例：破魔の理！）を画面中央にポップアップ表示。
- **被弾の震動 (Screen Shake)**: 痛烈な一撃、または術の発動時に画面全体を揺らし、物理的な衝撃を表現。
- **被弾の明滅 (Damage Flash)**: ダメージ時に画面が赤く閃光し、危機感を煽る。

### Phase 3: 【武勲の器】名刀と勲章の理（Loot & Medals）
- **武勲の品 (Rare Items)**: 敵調伏時に低確率で「名刀」「魔除けの札」「数珠」などをドロップ。
- **名品の言霊**: 入手時にその由来や物語をフレーバーテキストとして表示。
- **武勲の勲章 (UI Icons)**: 入手した品々が `CharacterCard` の隅に小アイコン（勲章）として刻まれ、隊員の歩みを可視化。

## 2. 実装の優先順位
1. **理の証明（自動検案の強化）**: 基盤が揺らがないようにするため最初に行う。
2. **術名のポップアップ**: 視覚的な変化を最も早く主殿（ユーザー）に届けるため。
3. **アイテム・勲章システム**: データの拡張（JSON）が必要なため、基盤整備後に行う。

## 3. 技術的負債の解消
- `App.jsx` 内の残存する純粋演算ロジックを `src/logic/` へ完全に抽出し、テスト可能にする。
- モバイル版の `dpad-btn` の連続タップ時の遅延やレスポンスをさらに改善。
