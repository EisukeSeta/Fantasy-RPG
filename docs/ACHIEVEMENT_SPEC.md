# ⛩️ 武勲システム仕様書：魂の刻印 (ACHIEVEMENT_SPEC)

## 1. 概要
武勲（称号/勲章）は、パーティ全体で共有される「栄誉（記録）」としての側面と、各隊員の魂に刻まれる「霊格（個別成長）」としての側面の双方を持つ。

## 2. 霊格 (Achievement Rank)
各隊員が得た武勲には「霊格（Rank）」が存在する。
- **取得**: 特定の条件（怪異調伏、特定イベント）で得られる権利。
- **成長**: 同種の称号を重複して得ることで、霊格が上昇する（最大 Rank 10）。
- **呼称**:
    - Rank 1: 拝領（基本補正発動）
    - Rank 2-9: 研鑽
    - Rank 10: 極致（特殊能力解放）

## 3. 共鳴判定 (Probability)
戦闘終了後に獲得可能な武勲に対し、隊員ごとに独立して取得判定が行われる。
- **基本取得率**: 20%
- **最低保証**: パーティの誰か一人は必ずその戦利（武勲）と共鳴する（全員が不採用となることはない）。
- **ボーナス**: その戦闘における貢献度（ダメージ量、回避回数、最後の一撃等）に応じ、各人の取得率が変動する（将来実装）。

## 4. 戦術的還元 (Effect)
各武勲が持つステータス補正（ATK, AC 等）は以下の式で算出される。
- **補正値** = `基礎補正値 * 霊格 (Rank)`
- **特殊効果**: Rank 10 に達した称号は、その者固有の「パッシブスキル」として永続的な特殊能力を授ける。

## 5. データ構造 (Data Structure)
各キャラクターオブジェクト内に `medals` 辞書を保持する。
```json
{
  "id": "abe_seimei",
  "medals": {
    "nue_slayer": 3,
    "guardian_of_shrine": 1
  }
}
```

## 6. 保存と永続性
- 図録（ArchivesView）の「獲得済みリスト」は、パーティの誰か一人が Rank 1 以上であれば開示される。
- 各人の霊格はセーブデータ（localStorage）に保存され、想起（Load）時に復元される。
