# ⛩️ 平安魔道伝 羅生門編：スマホ版継承 開発計画 (Mobile Handover Plan)

`HANDOVER_MOBILE_V1.4.md` および過去のプロジェクト計画を熟読し、PC版で達成された「データ駆動」「死の重み（HP/MP 1復活）」「美学の洗練」の理を、スマートフォン（モバイル）環境へ完全に適応させるための実装計画を策定しました。

PC版の意匠・仕様への影響を「最小限」に抑えつつ、モバイル特有のUXを向上させるための5つの段階に分けて実装を進めます。

---

## 🛠️ 第零段階：智の顕現（モバイル・デバッグの強化）
**課題**: スマホ実機やシミュレータでの検証を加速させるため、特定の状態（瀕死、全滅寸前、特定のイベント発生）を即座に引き起こす力が必要。
**実装内容**:
1. デバッグパネルに「瀕死（全員のHPを1にする）」ボタンを追加。これにより、第一段階で実装する `danger-overlay` の動作確認を容易にする。
2. モバイルの狭い画面でデバッグパネルが邪魔にならないよう、パネル自体の表示/非表示を切り替えるトグルボタンを導入。

## 🛠️ 第一段階：瀕死状態の視認性向上（UX調律）
**課題**: 復活後（HP1）の死地にあっても、モバイル版ではステータスが隠れがちであり危機感が伝わらない。
**実装内容**:
1. `index.css` に、HPが危機的状況（例：復活直後のHP「1」状態）であることを示す `danger-overlay`（赤黒く脈打つエフェクト）を追加。
2. モバイルのレイアウト（`.layout-mobile`）特有のミニステータスパネル（`.mini-status-panel`）にて、瀕死状態の際に背景色や文字色を赤く強く染めるなどの視覚的警告を付与。

## 🛠️ 第二段階：言霊（ログ）の操作性とモバイル特有のUI調整
**課題**: PC用に最適化された横スクロールログが、タッチ操作で快適か不安が残る。
**実装内容**:
1. `index.css` の `.log-content` 等に対し、モバイルでの滑らかなスクロールを保証する `-webkit-overflow-scrolling: touch;` を明記。
2. タッチデバイス用のスクロールバーの非表示・微調整を行い、フリック操作を快適に。PC版の表示（スクロールバーのカスタマイズ等）には影響させないよう分離。

## 🛠️ 第三段階：行軍の緊張感と操作性の担保（UI/UX向上）
**課題**: 復活地点(0,0)から社(1,1)への「命懸けの行軍」が、モバイルの小さな移動ボタンではミスタップ等のストレスになりうる。
**実装内容**:
1. `.move-btn` 等のタップ可能領域（padding）を調整し、誤タップを防ぐ極太の意匠へ強化。
2. タップ時の視覚的フィードバック（ボタンの沈み込みの強調や、アクティブ時の影の調整）を強化し、「重みのある一歩」を手元で感じられるようにする。

## 🛠️ 第四段階：技術的理（KPIとオーディオ制約）の検証と調整
**課題**: スマホ特有のブラウザ制約（自動ミュート）の検証と、タップ遅延を考慮したバランス検証。
**実装内容**:
1. `scripts/balance_tester.mjs` に、モバイル操作の遅延をシミュレートする処理（ターン処理に猶予を加算）を組み込み、モバイル想定の推定プレイ時間を算出する。
2. モバイル環境で `SoundEngine.js` が無音にならないよう、初回タップ（「はい/否」等のダイアログ）時に確実に Audio Context をレジューム（または初期化）させるロジックを確認し、必要であれば `App.jsx` を補強する。

---

## 📅 進行の取り決め
上記の「第一段階」から一つずつ順番に着手し、各作業の後に「モバイルシミュレーター（?mobile=1等）またはブラウザ検証」を実施します。
都度問題がないか報告・承認をいただきながら、段階的に実装を進めていきます。
