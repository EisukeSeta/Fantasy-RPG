## 2024-04-04: モバイル UI の情報集約とメッセージ領域の拡大

### 背景
モバイル画面の垂直スペースが不足しており、特に戦闘時にメッセージログが画面外に押し出される問題が発生。ダンジョン画面が固定サイズ（1:1）であるため、その内部スペースを有効活用する必要がある。

## 2026-04-04: PC版レイアウトの更なる洗練とロジックの統合 (v1.3)
- **現状**: 
    - 戦闘時にダンジョン画面が縦長になり、ボタンが画面外に押し出される（PC）。
    - ログの色分けが PC 版で不完全。
    - オープニングダイアログが PC 版で目立たない。
- **改善目標**:
    - **ダンジョン 1:1 保持**: `aspect-ratio` を用い、中央ペインのバランスを固定。
    - **ロジックの共用化**: コンポーネント抽出または共通関数を用い、PC/モバイル間の表示ルール（色など）を完全同期。
    - **PC オープニングの修繕**: ダイアログオーバーレイを全画面化し、没入感を向上。
    - **三ペインの拡張性**: 1:1 ダンジョンの左右の余白をサイドペインの拡張に活用。
    - ステータスバーが占めていた垂直スペースをメッセージログに割り当て、視認性を最大化する。
    - 全体レイアウトの安定化（崩れ防止）。

## [2026-03-31] タスク: lintエラーの全修正と自動コミット

### 背景
既存の `App.jsx` において、「Cannot call impure function during render」などの lint エラーおよび警告が発生している。これを解消し、プロジェクトの健全性を確保する。

### 実装プラン
1. **App.jsx のリファクタリング**
    - `useCallback` を導入し、`processMove`, `handleFight`, `handleRun` をメモ化する。
    - `useEffect` の依存配列に欠落している変数を追加し、最新状態へのアクセスを保証する。
    - インラインで行われている不純（impure）な呼び出しを整理し、Reactの純粋性を保つ設計にする。
2. **検証**
    - `npm run lint` を実行し、解決を確認。
    - `npm run build` を実行し、デプロイ可能な状態であることを確認。
3. **コミット**
    - 修復内容を記録し、Gitコミットを実行する。
