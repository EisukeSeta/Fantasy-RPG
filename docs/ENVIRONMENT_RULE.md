# ⛩️ 環境守護の理 (ENVIRONMENT_RULE.md)

本仕様書は、Windows 環境における平安幻想RPGの開発において、OSやツール仕様の差異による不具合（文字化け、ツール不在、誤翻訳）を未然に防ぐための絶対規範である。

## 第1章：言霊（エンコーディング）の守護
都のファイルはすべて UTF-8 (BOMなし) で統一される。

1. **PowerShell 操作の掟**
    - `Set-Content` や `Out-File` を使用する際は、必ず `-Encoding UTF8` を指定せよ。指定なき出力は「言霊の穢れ（文字化け）」を招く大罪である。
    - 例: `(Get-Content file.jsx -Encoding UTF8) | % { $_ -replace 'A', 'B' } | Set-Content file.jsx -Encoding UTF8`
2. **監査の掟**
    - 納品（奉納）前に必ず `git diff` を実行し、日本語コメントが `` や `ぁE` などの無残な姿に変わっていないか、一文字残らず目視せよ。

## 第2章：道具（コマンド仕様）の理
Windows (PowerShell) と Linux の道具の仕様差を正しく認識せよ。

1. **代替の法**
    - `grep` -> `Select-String`
    - `cat` -> `Get-Content`
    - `rm -rf` -> `Remove-Item -Recurse -Force`
    - `sed` -> PowerShell の演算子 `-replace` を活用せよ。
2. **存在証明の掟**
    - 外部ツール（`npx vitest` 等）を呼び出す際は、パスの通し方や Windows 特有のパーミッションに配意せよ。

## 第3章：不変（識別子）の掟
コード内の技術用語や識別子は、如何なる理由があろうとも翻訳の対象外とする。

1. **隔離の法**
    - 変数名（`activeBattlerIndex` 等）や CSS クラス、ID は翻訳・日本語化を一切禁ずる。
2. **置換の掟**
    - 大規模な置換を行う際は、単語境界 (`\b`) を使用した正規表現を用い、日本語文字列への意図せぬ干渉を防げ。

## 第4章：事故発生時の術式
不具合（文字化け、構文エラー）が発生した場合は、速やかに以下の手順を踏め。

1. **時を戻す（Revert）**: `git checkout -- file.jsx` で直ちに穢れを払え。
2. **原因究明**: どの操作（コマンド）が大地（環境）に背いたかを特定せよ。
3. **浄化**: 正しきエンコーディングとコマンドで再試行せよ。

---
**「大地を侮る者は、都を治める資格なし」**
本法典を遵守し、環境に依らぬ不変の安寧を築け。
