# ⛩️ 平安魔道伝 羅生門編：デバッグ機能基本仕様書 (DEBUG_SPEC.md)

都の検分を迅速化し、理の正しさを証明するための「智の術法」の定義。

## 1. 起動条件
- URL パラメータに `debug=true` を付与することで有効化。
- `const isDebug = ...` により、実行時に動的に理を切り替える。

## 2. 智のパネル（デバッグUI）
画面左下に浮遊するパネル。以下のボタンを備える。

| ボタン名 | 動作内容 | 用途 |
| :--- | :--- | :--- |
| **全快** | 全隊員のHP/MPを最大値に。状態を「平安」へ。 | 探索継続のテスト。 |
| **一撃** | 戦闘中、敵の現在HPを `1` に書き換える。 | 戦闘勝利後の遷移・レベルアップの検証。 |
| **消滅** | 戦闘を即時終了し、探索モードへ戻す。 | 敵遭遇をスキップしたい場合。 |
| **功徳** | 全隊員に一定の経験値（+1000等）を付与。 | レベルアップ演出、ステータス上昇の確認。 |
| **転移** | 任意の座標 (x, y) へ即座に移動。 | 特定イベント地点、ボス部屋への即時到達。 |
| **怪異召喚** | 任意の敵（またはランダム）をその場で発生。 | 特定の敵UI、戦闘ロジックの検証。 |

## 3. 隠された術（ショートカット）
- **壁抜け（Ctrl + 移動キー）**: 壁を無視して移動可能にする。
- **高速踏破（Shift + 移動キー）**: 移動時のメッセージや演出を短縮（または連打扱い）。
- **霧払い（Mキー）**: マップの全領域を `visited: true` にする。

## 4. 検証の自動化（今後の構想）
- **修羅の乱舞（自動探索テスト）**: AIがランダムに移動し続け、エラーが発生しないかを長時間回し続けるモード。
