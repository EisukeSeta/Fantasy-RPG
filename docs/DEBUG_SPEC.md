# ⛩️ 羅生門：デバッグ・検分基本仕様書 (DEBUG_SPEC.md)

本文書は、都の理（ロジック）を迅速かつ確実に検分するための「デバッグ機能」の絶対的な定義である。**本文書に記載のないデバッグ・パラメータや挙動はすべて「論理バグ」と見なす。**

## 1. 起動起動フラグ（URLパラメータ）
ブラウザ起動時に都の状態を制御する「神速の術式（フラグ）」を定義する。

| フラグ | 効果 | 位置づけ | 適用対象の State |
| :--- | :--- | :--- | :--- |
| **`debug=true`** | デバッグUI、チート機能の解禁。 | **【権限の法】** | `isDebugMode: true` |
| **`skip_intro=true`** | タイトル・オープニングを全スキップし、探索を開始。 | **【中略の法】** | `gameState: 'EXPLORING'`, `playerState: {x:0, y:0}` |
| **`start_battle=true`** | 導入を全スキップし、検証用の合戦を即時開始。 | **【強行合戦の法】** | `gameState: 'BATTLE'`, `enemy: [鼠僧正]`, `activeBattlerIndex: 0` |

### ⚠️ 廃止された理（禁忌）
- **`?seed=xxx`**: 複雑な初期化不整合の原因となるため、本プロジェクトより完全に廃止・抹消した。これをコード内で参照し続けることは論理バグである。

## 2. 初期化の優先順位（都の理の順序）
起動時、情報の重なり（競合）が発生した場合は以下の順序で状態を確定させる。

1.  **強行合戦 (`start_battle=true`)**: 
    - URLにこのフラグがある場合、最優先で「鼠僧正との一戦」を展開する。
    - 他のパラメータ、保存データはすべて無視される。
2.  **中略 (`skip_intro=true`)**: 
    - 合戦フラグがなく、このフラグがある場合、保存データを無視して「社（初期地点）」から開始する。
3.  **過去の記憶 (Save Data)**: 
    - 上記のスキップ・フラグが一切ない場合、`localStorage` からの復元を試みる。
4.  **天地開闢 (Default)**: 
    - 万策尽きた場合、タイトル画面（TITLE）を表示する。

## 3. 智のパネル（デバッグUI）
`debug=true` 時のみ画面左下に浮遊するパネル。詳細は `DEVELOPMENT_RULE.md` に準ずるが、以下の「転移のボタン」を必須とする。

- **[合戦：鼠僧正]**: クリックすると、`start_battle=true` と同じ状態を動的に注入し、即座に合戦を開始する。

## 4. 実装における遵守事項
- **文書先行**: いかなるデバッグ機能の変更も、まず本文書を更新し、主殿の承認を得てから行え。
- **神速の代償**: スキップ・フラグ使用時は、物語上必要なフラグ（イベントフラグ等）が未達になる可能性があるため、あくまで局所的なロジック検証に用いること。

## 5. 合戦開始時のアトミック契約 (Encounter Contract)
探索から合戦への遷移時（ランダムエンカウント、イベントエンカウント共）、以下の三つの状態書き換えは **「アトミック（不可分）」** に実行されなければならない。

1.  **敵の具現化**: `enemy` に有効なデータが注入されていること。
2.  **行動権の確定**: `activeBattlerIndex` が `null` でなく、最初の生存者にセットされていること。
3.  **世界線の移行**: `gameState` が `'BATTLE'` に変更されること。

**これらが分離して実行される、あるいは一つでも欠落した状態でレンダリングされることは論理バグであり、暗転（ブラックアウト）や進行不能の主因となる。**
