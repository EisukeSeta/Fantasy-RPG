# ⛩️ 実装計画 v3.0：平安魔道伝「都の鼓動」

本計画は、都の探索における情報視認性の向上と、合戦（戦闘）におけるドラマチックな演出の導入を目的とする。

## 1. 【千里眼】オートマップの強化
探索済みエリアが「霧の中に浮かび上がる」ような表現を目指す。

### 1.1 踏破マスの色彩調整
- **現状**: `#111` (ほぼ黒)
- **変更**: `#2c2c2c` もしくは 渋みのある茶系。さらに周囲に薄い光彩を追加。

### 1.2 壁の位置情報の露出
- **ロジック変更**: `processMove` 時、プレイヤーの周囲 1 マス（8 方向）にある `wall` セルも `visited: true` に設定。
- **描画変更**: `index.css` にて `.map-cell.wall.visited` の背景色を `#555` 程度に明るくし、未踏の壁（黒）と区別。

## 2. 【戦記の彩り】ダメージポップアップ（Floating Numbers）
合戦中、視線をログペインに移さずとも数値を直感的に把握できるようにする。

### 2.1 エフェクト・ステートの導入
```javascript
const [visualEffects, setVisualEffects] = useState([]);
// { id, targetId, value, type ('damage'|'heal'), timestamp }
```

### 2.2 表示位置の設計
- **敵（中央）**: 敵グラフィックの中央付近から発生。
- **味方（左/PC, 下/Mobile）**: 各キャラクターのステータスカード（HPバー付近）から発生。

### 2.3 CSS アニメーション
- `popup-up-fade`: 20px ほど上方にスライドしながら `opacity: 0` に向かう 800ms のアニメーション。
- ダメージは「赤・太字」、回復は「青・光彩付き」で表現。

## 3. 【震天動地】揺れと閃光
打撃の重みを視覚的に伝える。

### 3.1 画面の震撼 (Shake)
- 敵に痛打を与えた際、`pane-main` を微細に揺らす。
- 味方がダメージを受けた際、画面全体（`game-container`）を大きく揺らす。

### 3.2 霊力の閃光 (Flash)
- 術式発動時、背景を一時的に白または青にフラッシュさせる。
- クリティカルヒット（痛打）時、画面を一瞬赤く光らせる。

## 4. 実装ステップ
1. **[CSS]** 演出用クラス（`.popup-number`, `.flash-red`, `.shake-violent`）の定義。
2. **[Logic]** `App.jsx` への `visualEffects` ステート追加と自動消去（`setTimeout`）ロジックの実装。
3. **[Battle]** `handleFight` および `castSpell` 内でのエフェクト発行。
4. **[Map]** `processMove` の `visited` 拡張とスタイリング調整。

## 5. 考慮事項（スマホ版への影響）
- ポップアップ数字は、スマホ版の `mobile-status-dashboard`（ミニカード）上にも出るように CSS メディアクエリで調整。
- パフォーマンス維持のため、エフェクトの配列は最大 10 個程度に制限し、古いものから順次破棄。
