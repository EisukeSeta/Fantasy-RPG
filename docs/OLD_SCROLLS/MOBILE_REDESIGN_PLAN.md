# ⛩️ 平安魔道伝 羅生門編：スマホ版 UI/UX 抜本的再設計計画

現状の「上下二分割」レイアウトを廃止し、**「ダンジョン視界を主軸とした全画面没入型レイアウト」**へと昇華させます。

---

## 🛠️ 設計思想：没入感の極大化
スマホの縦長画面を最大限に活かし、プレイヤーが常に「都の景色」の中にいる状態を作ります。

1.  **ダンジョン視界の全画面化**: 迷宮のワイヤーフレーム（`view-window`）を背景とし、他のUI要素はすべてその上に浮かぶ「透過オーバーレイ」として配置します。
2.  **情報の層（レイヤー）構造**:
    - **最背面**: ダンジョン視界（`view-window` / `WireframeView`）
    - **中間面**: 戦闘時の敵怪異表示（`pane-enemy`）、メッセージログ（透過）、移動ボタン（透過）
    - **最前面**: 全画面オーバーレイ（迷宮図、隊員之証、ダイアログ）
3.  **既存データ・ロジックへの影響遮断**:
    - `App.jsx` の構造は極力維持し、CSS（`index.css`）のメディアクエリ設定を中心に刷新します。
    - PC版への影響を避けるため、`.layout-mobile` スコープ内でのみ定義を上書きします。

---

## 🛠️ 実装の具体策

### 1. ダンジョン視界の拡張
- `.layout-mobile .pane-main` を `height: 100vh`（または `100dvh`）とし、画面全体を占有させます。
- `aspect-ratio: 1 / 1` を解除しつつ、描画領域（`canvas`等）自体は中央付近に大きく保ちます。

### 2. 透過オーバーレイ UI の導入
- **浮遊移動盤 (Floating Move Buttons)**:
    - 画面下部に透過度の高い（`rgba(0,0,0,0.5)`程度）移動ボタンを配置。
    - 前進・後退・回転を十字キー状、またはコンパクトなグリッドにまとめ、ダンジョン視界を遮らないようにします。
- **言霊の窓 (Translucent Log)**:
    - ログ表示を画面下部または中央付近に浮かぶ透過パネルに変更。
    - 最新の 3〜5 行程度を表示し、スクロール操作で過去を辿れるようにします。
- **ミニステータス**:
    - 画面上部またはボタン付近に、HP/MPバーのみを細く配置。

### 3. モード切替の洗練
- 戦闘開始時、敵の御名と姿がダンジョン中央にドロップシャドウを伴って大きく浮かび上がるように調整。
- 探索/戦闘の切り替えをアニメーション（フェード等）で行い、画面の切り替わりを滑らかにします。

---

## 📅 進行手順

1.  **[設計の検証]**: 透過オーバーレイにした際の視認性（とくに暗いダンジョン内でのボタンの見え方）をブラウザエージェントで事前確認。
2.  **[CSS刷新]**: `index.css` の `.layout-mobile` 関連セクションを全面的に書き換え。
3.  **[App.jsx 調整]**: モバイル時のレンダリング順序や、透過パネル内の文字色の調整。
4.  **[最終浄化]**: `isForceMobile` 時の操作感が「平安の闇」を損なっていないか実機（シミュレータ）で検分。

---

## 💡 アドバイス・確認事項
- **アスペクト比の維持**: 3Dワイヤーフレーム（`WireframeView`）はアスペクト比が崩れると歪むため、描画領域自体は正方形を維持しつつ、その周囲（背景）を美しく配置する工夫を凝らします。
- **ボタンの押しやすさ**: 全画面化すると、ボタンが指の影に隠れがちになります。右利き/左利きを問わない中央配置や、好みの位置に配置可能なスペースを確保します。
