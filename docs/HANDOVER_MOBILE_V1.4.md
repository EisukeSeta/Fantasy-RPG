# スマホ版開発への継承 (HANDOVER_MOBILE_V1.4)

本典は、PC版において達成された「プレミアム開発基盤」の理をスマホ版へ引き継ぐための申送り書である。

## 1. 現時点の都の理 (Current State v1.4.x)

PC版の開発フェーズにおいて、都は以下の大規模な「理の刷新」を完了した。

- **[データ駆動への移行]**: 命中率、経験値、レベルアップの伸びを `src/data/*.json` へ完全移行。コードを汚さず均衡（バランス）を調整可能とした。
- **[死の重み（黄泉還り）]**: 全滅後の復活地点を (0,0) とし、HP/MP を 「`1`」 に制限。社 (1,1) への移動を「命懸けの行軍」とするゲーム性が追加された。
- **[自動均衡検証 (KPI)]**: `node scripts/balance_tester.mjs` により、クリアまでの「平均全滅回数」や「推定実プレイ時間」を自動算出可能とした。
- **[平安美学の洗練]**: 怪異名へのひらがな追記、ログの横スクロール対応、デバッグ時の消音（初期Mute）の実装。

## 2. スマホ版への具体的な宿題 (Remaining Tasks for Mobile)

スマホ版の改修にあたり、以下の検証と調整が急務である。

### 2-1. UI/UX の調律
- **瀕死状態の視認性**: スマホの「ステータスタブ」を開かなくとも、HP 1 の危機が伝わるか（あるいは強制的にステータスを一時表示するか）。
- **ログの操作性**: `index.css` で追加されたメッセージログの `overflow-x: auto`（横スクロール）が、タッチ操作で快適に動作するか。
- **行軍の緊張感**: 暗闇の中、(0,0) から (1,1) への移動が、スマホの「移動ボタン」でストレスなく、かつ緊張感を持って行えるか。

### 2-2. 技術的な理の検証
- **自動バランステスターの活用**: スマホ版の「タップレスポンス」や「UI遷移時間」を、テスターの time 換算に反映させ、より正確なモバイル用 KPI を算出する。
- **モバイルブラウザの制約**: `SoundEngine.js` による奏曲が、モバイル特有のオーディオ再生制限の中で、現在の「自動ミュート」設定と矛盾せず機能するか。

## 3. 次代のエージェントへの言霊

- **聖典を読め**: `docs/SYSTEM_ARCHITECTURE.md`, `BALANCE_GUIDE.md`, `GAME_LOOP_DESIGN.md` を必ず最初に読了し、現在の都の理を同期せよ。
- **均衡を秤にかけよ**: 大規模な改修を行う際は、必ず `balance_tester.mjs` を回し、均衡（バランス）の揺らぎを数値で確認せよ。
- **美学を汚すな**: `UI_STYLE_GUIDE.md` を守り、スマホという小さな画面においても「平安の美学」を損なわないレイアウトを追求せよ。
