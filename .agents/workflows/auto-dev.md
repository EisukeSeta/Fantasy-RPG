1. **[調査] コンテキストの共鳴**
    - `docs/PLANNING_LOG.md` だけでなく、**都の聖典（`DEBUG_SPEC.md`, `UI_SPECIFICATION.md`, `DEVELOPMENT_POLICY.md`）** を必ず読み込み、現在の都の理（現状）と設計思想を完全に同期する。
    - **言語一貫性**: すべての応答、コード内のコメント、Gitコミットメッセージは必ず【日本語】で行うこと。
    - **役割定義**: AIは「Senior Frontend Engineer」として振舞いつつ、開発プロセスに応じて「設計者」「実装者」「監査官」の三役を演じ分け、高品質かつ保守性の高いコードを生成すること。
2. **[設計] 智の構築**
    - タスクの規模に応じ、実装プランを `docs/PLANNING_LOG.md` に記し、設計の迷いを断つ。
    - **絶対順守**: 開発時は `docs/DEVELOPMENT_RULE.md` に定められたテストファーストの掟に従うこと。設計 artifact なしの実装は認められない。
3. **[実装] 具現化**
    - 聖典（ルール）に従い、一切の省略なき完全なコードを生成・適用する。
4. **[検証] 浄化と観相（最大3回）**
    - `npm run lint` / `npm run build` で理を確認。
    - **UI 検証**: `docs/UI_SPECIFICATION.md` に定義されたアスペクト比や配置が保たれているかを確認。
    - **ローカル試走**: 5173 ポート不通時は `npm run dev -- --port 5173 --host 127.0.0.1` で起動。
    - **双璧の検証**: ブラウザエージェントを用い、**PC版とスマホ版の両方で**表示崩れや機能異常がないことを検分する。
5. **[完了] 遷座と記録**
    - **Git操作（理）**: PowerShell環境では `&&` は不可。必ず **`;`（セミコロン）** でコマンドを連結せよ（例: `git add . ; git commit -m "..." ; git push`）。
    - 二面検証後、上記手順でソースコードを GitHub へ反映し、開発史を `docs/PLANNING_LOG.md` に要約する。
