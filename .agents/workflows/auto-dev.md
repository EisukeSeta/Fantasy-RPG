// turbo-all

1. **[調査] コンテキストの把握**
    - `docs/PLANNING_LOG.md` と関連ソースコードを読み込み、現在の状況を特定する。
2. **[設計] プランニング（規模に応じて）**
    - タスクが大規模な場合のみ、実装プランを作成し `docs/PLANNING_LOG.md` に追記する。
3. **[実装] コード生成**
    - ルールに従い、省略なしの完全なコードを出力する。
4. **[検証] 自動テストと修復（最大3回）**
    - `npm run lint` や `npm run build` を実行。エラーがあれば自律的に修正。3回失敗した場合は停止し、人間に報告する。
5. **[完了] コミットと報告**
    - 正常にビルドできたら `git commit` を実行し、完了を日本語で報告する。
