// turbo-all

1. **[調査] コンテキストの共鳴と開発会議**
    - `docs/PLANNING_LOG.md` だけでなく、**都の聖典（`DEBUG_SPEC.md`, `SYSTEM_ANALYST_LOG.md`, `DEVELOPMENT_RULE.md`）** を読み込み、現状を同期する。
    - **エージェント協業**: 調査結果を受け、四役（設計・実装・監察・計量）による短い会話形式の分析を表明し、主殿に状況を報告する。
2. **[設計] 智の構築（合議制）**
    - タスクの規模に応じ、実装プランを `docs/PLANNING_LOG.md` に記す。
    - 各役割からの「設計上の懸念・期待」を対話形式で提示し、主殿の承認を得る。
    - **段階的実装の掟**: 大規模な機能追加時は一気に実装せず、必ず TODO リスト（細分化されたマイルストーン）を作成し、一つ一つの工程を主殿に確認・報告しながら着実に進めること。
    - **絶対順守**: 開発時は `docs/DEVELOPMENT_RULE.md` と **`docs/DATA_CONTRACTS.md`** に定められた掟と契約に従うこと。設計 artifact なしの実装は認められない。
3. **[実装] 具現化**
    - 聖典（ルール）に従い、一切の省略なき完全なコードを生成・適用する。
    - **守備範囲の掟**: 一度の修正は原則一モジュール（＋そのテスト）に限定せよ。
4. **監査と検証 (Audit & Verify)**:
    - 実装後、監査官として `npm run test` 等の自動検証、Build、Lint を実行すること。
    - **UI 検証**: `docs/UI_SPECIFICATION.md` に定義されたアスペクト比や配置が保たれているかを確認。
    - **ローカル試走（不動の門）**: サーバー起動時は必ず **5173 ポート** を指定せよ。もしポートが塞がっている場合は、既存プロセスを特定・停止し、5173 での再起動を完遂すること。プレイヤーを他のポートへ迷い込ませてはならない。
    - **強制浄化**: キャッシュによる不具合が疑われる際は `npm run dev -- --force --port 5173` を実行せよ。
    - **双璧の検証**: ブラウザエージェントを用い、PC版とスマホ版の両方で表示崩れや機能異常がないことを検分する。
    - **神速の行使**: ブラウザ検証時は必ず **`docs/TEST_SPECIFICATION.md`** を参照し、URLパラメータ（seed）を活用して目的の状態（図録、ボス戦等）へ即座にアクセスすること。プロローグや移動の反復（トークンの浪費）は禁忌である。
5. **[照合] 奉納前の儀式 (Diff Audit)**
    - **git diff の実行**: 奉納前に必ず `git diff` および `git diff --staged` を実行し、変更点を詳細に読み取ること。
    - **監査報告**: 「削除されたプロパティの有無」「意図せぬコード混入がないこと」を主殿に報告し、承認を得ること。
6. **[完了] 遷座と記録**
    - **Git操作（理）**: PowerShell環境では `;`（セミコロン）でコマンドを連結。
    - 承認後、GitHub へ反映し、開発史を `docs/PLANNING_LOG.md` に要約する。
